## Project Overview

**Western Tech Hub** is a fully responsive, multi-page business website built to demonstrate mastery of modern version control practices using Git and GitLab. The site represents a fictional web development company and was developed from scratch while applying professional branching, committing, merging, and collaboration workflows throughout the entire development lifecycle.

This project proves two distinct skills simultaneously: the ability to build a real, functional web product AND the discipline to manage that build through a structured version control process — the same workflow used in every professional engineering team.

## Version Control Skills Demonstrated

This project was not just about building the website — it was about managing the entire development process like a professional engineering team:

- **Repository initialization** — Set up a GitLab remote repository from scratch, connected it to a local environment, and established the project structure.
- **Branching strategy** — Created feature branches for each page and component, keeping the `main` branch stable at all times.
- **Commit discipline** — Made frequent, descriptive commits that document the progression of work — not bulk saves, but intentional checkpoints tied to completed features.
- **Merge requests** — Used GitLab Merge Requests to simulate a code review workflow before merging feature branches into `main`.
- **Conflict resolution** — Resolved merge conflicts that arose when multiple branches modified shared CSS and navigation files.
- **Remote collaboration workflow** — Practiced pull/fetch/push cycles to simulate working on a shared codebase with a distributed team.

Every software engineering role — from junior developer to staff engineer — requires Git fluency. This project demonstrates that I don't just know Git commands; I understand *why* version control exists and how to use it to protect code quality, enable team collaboration, and maintain a clean project history.

The habits built here — atomic commits, branch-per-feature, merge request reviews, never committing directly to main — are the same habits that prevent production outages, make code reviews efficient, and allow teams to ship software with confidence.

- Pull request / merge request culture
